SUPPORT ME! - Documentación

Base URL: /, Version: 0.1

Default request content-types: application/json
Default response content-types: application/json
Schemes: HTTP

Resumen del API

Ruta Operacion Authorization Descripcion
/categories GET Retorna todas las categorias de las huecas registradas en la base de datos
/category/<int:pk> GET Retorna la categoria identificado por el id especificado en pk, en la url, realizando un filtro en los registros de la base de datos
/cities GET Retorna todas las ciudades registrados en la base de datos
/city/<int:pk> GET Retorna la ciudad identificado por el id especificado en pk, en la url, realizando un filtro en los registros de la base de datos
/comment GET Token Guarda en la base de datos, un comentario realizado por un usuario en una hueca
/comments/<int:hueca> GET Retorna todos los comentarios realizados en la hueca recibido como parametro en la url
/comment/<int:pk> GET Token Elimina un comentario de un usuario en una hueca
/hueca POST Token Guarda la informacion de la hueca obtenido en el paquete recibido en la base de datos
/hueca/<int:pk> GET Responde con un paquete conteniendo la informacion de la hueca encontrada por el parametro pk en el servidor
PUT Token Guarda la informacion de la hueca en la base, identificado por pk
DELETE Token Elimina la hueca identificada por el pk de la base de datos
/huecas GET Responde con un paquete que almacena la informacion de todas las huecas registradas en el servidor
/huecas/categories/<int:category> GET Retorna todas las huecas que cumplan con la categoria recibida como parametro en category
/huecas/cities/<int:city> GET Responde con un paquete que contiene todas las huecas asociadas al identificador de la ciudad recibido en el requerimiento
/huecas/search/<str:search> GET Retorna un listado de huecas cuyo nombre coincidan con el parametro search, enviado como parametro
/location/<str:latitude>/<str:longitude>/<str:km> GET Responde con un paquete que contiene las huecas que se encuentren cerca de la ubicacion(latitud, longitud) y km.
/image POST Token Almacena la imagen enviada en el paquete en el servidor
/images/<int:hueca> GET Retorna todas las imagenes de la base de datos que esten relacionadas con el identificador hueca
/image/<int:pk> GET Retorna la imagen identificada por pk de la base de datos
DELETE Token Elimina la imagen identificada por pk de la base de datos, retorna una respuesta de eliminacion exitosa
/menu POST Token Recibe la informacion de un plato de comida enviado en el paquete y lo guarda en el servidor
/menus/<int:hueca> GET Retorna todos los platos de comida guardados en el servidor y que estan asociados con el parametro hueca
/menu/<int:pk> GET Retorna un plato de comida de la hueca, el plato es identificado en la base por el parametro pk
DELETE Token Elimino un plato de comida de la hueca, el plato a eliminar es identificado en la base por el parametro pk
/like POST Token Guarda el registro de un like de un usuario en la hueca
/likes/<int:hueca> GET Retorna una listado de los usuarios que han dado like a la hueca recibido como parametro
/like/<int:hueca>/<int:user> GET Token Retorna un like realizado por el user en la hueca, ambos datos son recibidos de la url
DELETE Token Elimino un like realizado por el user en la hueca, ambos datos son recibidos de la url
/likes/<int: user> GET Token Retorna un listado de las huecas que el usuario le ha dado like
/rating POST Token Guardo una calificacion hecha por un usuario en una hueca en el servidor
/ratings/<int:hueca> GET Regresa las calificaciones hechas por los usuarios en la hueca recibido como dato en la url
/rating/<int:hueca>/<int:user> GET Token Regresa el listado de todas las calificaciones realizadas por el usuario definido en user, en la hueca recibida como dato en la url
/user/<int:pk> GET Token Retorna un usuario
/login POST Valida las credenciales de inicio de sesion enviadas por el usuario, y retorna una respuesta con el token y datos del usuario en caso de validacion exitosa
/signup POST Registra un nuevo usuario en la aplicacion, donde se valida si las credenciales enviadas son las correctas para realizar el registro
/logout/<int:pk> DELETE Token Elimina el Token del usuario que ha iniciado sesión.
/update/<int:pk> PUT Token Actualiza el Usuario.
/delete/<int:pk> DELETE Token Elimina el Usuario o simplemente lo desactiva.
/users GET Token Retorna todos los usuarios registrados en la aplicación
